About Application About

This static web application was developed using the Django framework to demonstrate and practice coding skills in the context of web development using Python. The source code was developed over a period of six weeks, although some of the files that can be accessed in the app were compiled over a period of five years. The application was built in Visual Studio code on a Linux operating system and tested in Firefox for Linux. Coding languages used include Python and Javascript, as well as the markup and styling languages, HTML, and CSS. The source code can be found in this Github repository.


Most of the time was taken in learning about the Django framework, revisiting Javascript coding techniques, and adjusting the dimensions to make the website responsive. The lessons and experience from building this application will significantly reduce the amount of time taken to finish a similar project. Overall, experience from coding other programs in Java, C, Kotlin and Python expedited production as in the case where the knowlege of program variables, computer memory, program functions and call stacks, and that the learning curve can be sharp, spurred the project to completion. This about page serves to give an informal documentation of the application and the some of the sections of the code that stood out as challenging or interesting. Code is included in images as an illustration of the topics discussed such as Django's views, models and settings modules which are used to handle HTTP requests and responses, define entities in the app and configure the app, respectively. Sources related to building Django web applications are included at the bottom of the page.


Each web page in the application is associated with a view in Django. In short, a view is a Python function which processes HTML templates. Each view performs certain tasks, such as the 'contact' view which captures information added to the form after the send button is clicked and transmits the notification email to the web developer. The view module handles the http requests and responses to generate a page from the HTML. Javascript functions control the features on each page, such as selecting which menu to show based on the viewport width. Together, the Python and Javascript functions ensure that the application functions as intended. Django also offers the use of the model feature, which takes ideas from object-oriented programming in that it uses classes to describe entities. Theses models can be found in the models.py module. Django constructs admin tables to more easily handle data. For example, the portfolio model declares files and file names that are associate with a project. The files come in two type of compression. The programmer writes the code in Python and Django creates a secure admin page containing all the information.


The settings module in Django is arguably one of the things that make the learning the framework worthwhile. This is because the settings.py file contains configuration description required for the application to work. It tells the web browser where to look for certain instructions and handles interactions with external APIs and API keys. Django describes the settings module as a "Python module with module-level variables". Items like static folders and paths containing all the images, CSS files and scripts are instantiated in the settings module and their variables name can be used anywhere in the app.

The part of the project that took the most time was making the app responsive. A website is responsive if it can change its appearance based on the dimensions of the viewport. Nowadays, it is an unwritten rule that a website should be responsive. Another unwritten rule is that designers and developers should use the mobile-first approach which focuses on building a website for small to medium sized screens. This website however, did not follow that rule. The website for larger viewports was designed and coded first, and thereafter, pixels were changed to percentages of the viewport width and height to ensure a uniform design approach across different devices. Fine tuning the percentages is much like approximating an integral using the sum of areas across small intervals, where the intervals are tantamount to the width intervals. Defining the appearance of a page across smaller width intervals gives a smoother response and makes the website more customisable for the developer.

Developing a website takes time. Every detail needs to be assessed, from the distance between images, font-size, when things should be displayed to the screen and most importantly, how data retrieved from a user should be processed. One can think of developing a web app like creating a brand new planet and how things look and work depends entirely on the imagination of its creator. To reduce the amount of time taken to generate a web app, some of the best practices that can be implemented include:


  • Code templates for most of your work. Do not repeat code or the logic in code
  • Use Javascript instead of CSS to control most features related to appearance
  • Import frameworks that handle the responsiveness of the app instead of using @media excessively
  • Use global variables in scripts instead of session storage to store values that need to be reused in the app
  • Do not be reluctant to change images, text or the position of items on the screen to make the web page responsive while maintaining the same design theme throughout the app
  • Use short but discernable variable names and comment code

Up next should be building more advanced web applications which implement complex algorithms and APIs. The idea is to use optimisation techniques to create an AI-based product where users can automate daily tasks to improve their living standards.
